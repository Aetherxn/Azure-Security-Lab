# Design and Secure Azure Network Infrastructure
## Objective

The aim of this repo is to secure Azure network access with Network security groups (NSGs) and Application security groups (ASGs) by isolating the environment that connects both devices & servers and segmentating the networks into smaller isolated sub-sections. Combining both of these techniques greatly improves network security by limiting the blast radius of cyberattacks reducing network congestion and simplifying compliance.

## Architecture Decisions

- Use a virtual network with a single subnet to isolate, organise, and secure cloud and on-prem resources in a simple layout.
- Create Application Security Groups (ASGs) to group similar workloads and apply more granular traffic control based on application needs.
- Attach Network Security Groups (NSGs) at subnet level to keep security rules consistent across all VMs and strengthen overall network protection.
- Use NSG rules as a built-in traffic filter to control inbound and outbound access and improve security across cloud resources.

## Key Learnings

- Built a secure Azure network using VNets and Subnets to isolate resources and improve network segmentation.
- Configured NSGs to control inbound and outbound traffic with rule-based access controls.
- Used ASGs to simplify security management by applying policies to groups of VMs based on their function.
- Applied NSGs at the subnet level to enforce consistent security policies across workloads.
- Reduced the attack surface and limited potential lateral movement by separating resources into isolated network segments.
- Gained hands-on experience with Azure networking, traffic filtering, and network security best practices.
