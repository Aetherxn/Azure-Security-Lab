# Design and Secure Azure Network Infrastructure
## Objective

The aim of this repo is to secure Azure network access with Network security groups (NSGs) and Application security groups (ASGs) by isolating the environment that connects both devices & servers and segmentating the networks into smaller isolated sub-sections. Combining both of these techniques greatly improves network security by limiting the blast radius of cyberattacks reducing network congestion and simplifying compliance.

## Architecture Decisions

- Use Virtual network with one subnet to isolate, organise and secure the cloud and on-prem computing resources.
- Creating ASG's allows us to have granular control over traffic within our network by creating detailed network security policies based on our workloads, applications, or environments.
- Associating NSGs to Subnets keeps security policy consistant across all VMs within that subnet and protects the subnet from potential threats by restricting access to it.
- Implementing NSG rules acts as a built-in firewall to secure cloud resources by filtering inbound and outbound traffic

## Key Learnings

- **Virtual network** is a logically isolated network within the azure cloud where resources (VM's,Databases and applications) can securely communicate.
- **Subnets** are smaller isolated subsections of a virtual network that allow admins to organise, secure and manage resources efficiently.
- **Network security groups** are a key component of Azure Networking that control inbound and outbound traffic to resources. NSGs act as built-in virtual firewalls, enabling admins to define rules based on IP addresses, ports, and protocols.
- **Application security groups** simplify security rule management by grouping VMs based on their role. This allows us to apply security rules to groups of VMs rather than individually.
- **Subnet Association** keeps security policy consistant across all VMs within that subnet.
